Version Control System


Version Control System (VCS) merupakan sebuah kumpulan perangkat lunak yang terintegrasi dan digunakan untuk membantu software engineer mengelola perubahan source code dari waktu ke waktu. Dengan perangkat lunak ini sistem mampu melacak perubahan atau modifikasi source code. Jika suatu waktu ditemukan kesalahan akibat human error maka software engineer dapat dengan mudah membandingkan dengan versi sebelumnya dan melakukan perbaikan.


Cara Kerja Version Control System


Version control (kontrol versi) menggunakan repository atau repositori (database perubahan) dan Salinan pekerjaan di mana kita melakukan pekerjaan. Salinan kerja (kadang-kadang disebut checkout dalam istilah pemrograman) adalah salinan pribadi dari semua file dalam proyek. Kita dapat mengedit salinan tersebut, tanpa memengaruhi rekan satu tim. Saat perubahan dirasa sudah sesuai maka dapat dilakukan komit (commit) perubahan ke repositori. Repositori merupakan database dari semua pengeditan untuk, dan atau versi historis (snapshot) dari, proyek. Repository mungkin berisi perubahan yang belum diterapkan ke copy pekerjaan projek lokal. Kita dapat memperbarui salinan respository (pull) sesuai dengan perubahan terbaru yang ada di respository. Secara sederhana repository merupakan database perubahan yang menampung riwayat atau sejarah linier dari setiap perubahan yang telah dilakukan. Karena repository menyimpan riwayat perubahan dan setiap software engineer yang terlibat dalam projek bisa membuat salinan dari repository maka kita dapat melakukan perubahan secara bersamaan (kolaboratif) dan menggabungkannya (branch dan merge). Dengan adanya riwayat perubahan juga mempermudah pelacakan (trace) dari kode yang sedang dikerjakan.
